A whimsical gallery showcasing how different AI models interpret and render the prompt: _"a pelican on a bicycle"_.

This project provides a visual snapshot of the evolution of AI image generation capabilities across various models from Anthropic (Claude) and OpenAI (GPT), demonstrating how different models approach the same creative challenge.

## What is this?

A Next.js gallery application that displays SVG renderings of pelicans on bicycles, each generated by different AI models. It's a fun, visual way to explore how various LLMs interpret and visualize the same prompt over time.

### Features

- **Search & Filter**: Search by model name or provider with debounced input
- **Randomize**: Get random SVG selections from specified model groups
- **Show All**: View all SVGs grouped by provider
- **Responsive Grid**: Beautiful card-based layout with loading states
- **Database-Backed**: PostgreSQL database with Prisma ORM for efficient data management

## Getting Started

### Prerequisites

- Node.js (v18 or higher)
- npm, pnpm, yarn, or bun
- PostgreSQL database

### Installation

1. Clone the repository

2. Install dependencies:

```bash
npm install
# or
pnpm install
```

3. Set up your database:

Create a `.env.local` file in the root directory:

```env
POSTGRES_URL="postgresql://user:password@localhost:5432/pelicans_demo"
```

4. Run database migrations:

```bash
npm run db:migrate
```

5. Seed the database:

```bash
npm run db:seed
```

6. Run the development server:

```bash
npm run dev
```

7. Open [http://localhost:3000](http://localhost:3000) in your browser

## Architecture

The project follows a service layer architecture:

- **Service Layer** (`app/lib/svg-service.ts`): Contains Prisma queries and database logic
- **Actions Layer** (`app/actions.ts`): Server actions that wrap service layer functions for client components
- **Client Components**: React components that call server actions

This separation ensures clean code organization and proper separation of concerns.

## Database Schema

The database consists of three main models:

- **Provider**: AI model providers (e.g., Anthropic, OpenAI)
- **Model**: Specific AI models (e.g., Claude Sonnet 3.7, GPT-4.1)
- **Svg**: SVG content generated by models
